def comp_interest(series: pd.Series, amount: float) -> float:
  '''Recebe uma pandas.Series com valores em porcentagem e um capital aplicado,
     e retorna o valor acumulado aps o decorrer da srie.'''
  return round(amount * (1 + series.mean()/100) ** len(series), 2)

def date_value(series: pd.Series, sel: list) -> dict:
    '''Recebe uma pandas.Series e uma seleo que deve retornar apenas 1 item da mesma,
       e retorna a combinao do ndice (data) convertido em texto e valor do item
       arredondado num dicionrio com duas chaves.'''
    if sel.sum() != 1: raise Exception("'sel' must return only one True")
    date = series[sel].index.strftime('%m/%Y').astype(str)
    value = round(series[sel], 2)
    out = dict()
    for (date, value) in zip(date, value): out['idx'] = date; out['val'] = value
    return out

class lm:

    def __init__(self, data, y_var):
        # data prep
        self.data = data.dropna()
        self.y = self.data[[y_var]].to_numpy()
        self.x = self.data.loc[:, self.data.columns != y_var].to_numpy()

        # split data
        self.x_train, self.x_test, self.y_train, self.y_test = train_test_split(
            self.x, self.y, test_size = 0.25)

        # model evalution
        self.lm = LinearRegression().fit(self.x, self.y)
        self.lm_train = LinearRegression().fit(self.x_train, self.y_train)

        # metrics
        self.y_pred = self.lm.predict(self.x)
        self.y_pred_train = self.lm_train.predict(self.x_test)

        self.residuals = self.y_pred - self.y
        #self.residuals_train = self.y_pred_train - self.y_train

        self.coef = dict(
            coefficients=self.lm.coef_.tolist(),
            intercept=self.lm.intercept_.tolist())
        self.coef_train = dict(
            coefficients=self.lm_train.coef_.tolist(),
            intercept=self.lm_train.intercept_.tolist())

        self.r_sqr = self.lm.score(self.x, self.y)
        self.r_sqr_train = self.lm_train.score(self.x_test, self.y_test)

        self.mae = mean_absolute_error(y_true=self.y, y_pred=self.y_pred)
        self.mae_train = mean_absolute_error(
            y_true=self.y_test, y_pred=self.y_pred_train)

        self.mse = mean_squared_error(y_true=self.y, y_pred=self.y_pred)
        self.mse_train = mean_squared_error(
            y_true=self.y_test, y_pred=self.y_pred_train)

        self.rmse = mean_squared_error(
            y_true=self.y, y_pred=self.y_pred, squared=False)
        self.rmse_train = mean_squared_error(
            y_true=self.y_test, y_pred=self.y_pred_train, squared=False)

    def params(self, train=True, deep=True):
        if train: return self.lm_train.get_params(deep=deep)
        else: self.lm.get_params(deep=deep)

    def predict(self, x, train=True):
        x = pd.DataFrame(x).to_numpy()
        if train: out = self.lm_train.predict(x)
        else: out = self.lm.predict(x)
        return pd.DataFrame({'pred': out[:,0]})

2.2. Encontrando os juros reais

Adicionando juros real, usando a frmula de Fisher:

$r=\frac{(1+i)}{(1+\pi)}-1$

Onde $i$ so os juros nominais, $\pi$ a inflao, e $r$ os juros reais.

Muito interessante como estas duas variveis parecem estar relacionadas! Mas existem algumas coisas que explicam este fato:

Uma delas a forma com estas sries foram construdas. Voc deve lembrar que no tpico 2.3 eu usei a funo cumsum() para construir ambas as funes. Uma maneira de entender o que esta funo faz, que ela "emplilha" o prximo valor em cima da pilha de todos os outros valores anteriores, ento pelo simples fato de a maioria dos valores das sries originais serem positivos (como vimos em 2.1), temos uma linha crescente como resultado em ambos os casos.

Mas outra coisa que podemos usar como explicao tambm a funo da taxa base de juros. Como eu disse anteriormente, o CDI sempre acompanha a SELIC, e o Conselho Monetrio usa essa taxa para controlar a inflao, consequentemente, ele deve reagir ao nvel atual de inflao definindo uma nova taxa de juros (juros altos em tempos de inflao alta). Como resultado, temos uma curva de juros real modesta, mas na maior parte do tempo positiva (como vimos em 2.2).

3.1. Tempo de resposta da inflao

Agora que sabemos como a inflao e a taxa de juros esto relacionadas entre si, uma coisa interessante que podemos fazer tentar medir quanto tempo deve passar at que a inflao comece a responder mudana da taxa de juros.

A maneira simples de fazer isso atrasar uma srie em relao outra at que uma delas at conseguir a maior correlao possvel. O tempo de atraso aplicado ser o tempo de reao esperado para uma mudana na taxa de juros.

3.2. Prevendo a inflao futura com os juros presentes

Agora que j temos uma boa idia de quanto tempo ser necessrio para que a inflao reaja taxa de juros, podemos tentar prever a inflao nos prximos meses!

Voc j deve estar cansado desse texto, mas o que temos no final vai compensar a leitura. J que a taxa de juros de meses atrs pode ajudar a entender a inflao de hoje, podemos usar a taxa de juros de hoje para tentar entender a inflao do futuro! Vamos comear treinando um modelo de regresso, o nosso modelo ter este formato:

$$ \pi_{(t=14)} = \alpha \times i_{t=0} + \beta $$

Significa que usaremos os estimadores taxa de juros $i$ no momento $t=0$ para prever a inflao $\pi$ no momento $t=14$. $\alpha$ e $\beta$ vo nos ajudar a ajustar a curva de regresso no lugar certo, estes so os nossos parmetros.

O motivo de usar a inflao adiantada, ter a oportunidade de treinar o modelo com os dados mais recentes de inflao, o que muito importante quando se trata de realizar previses para o futuro (forecasting).
