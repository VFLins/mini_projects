def comp_interest(series: pd.Series, amount: float) -> float:
  '''Recebe uma pandas.Series com valores em porcentagem e um capital aplicado,
     e retorna o valor acumulado após o decorrer da série.'''
  return round(amount * (1 + series.mean()/100) ** len(series), 2)

def date_value(series: pd.Series, sel: list) -> dict:
    '''Recebe uma pandas.Series e uma seleção que deve retornar apenas 1 item da mesma,
       e retorna a combinação do índice (data) convertido em texto e valor do item
       arredondado num dicionário com duas chaves.'''
    if sel.sum() != 1: raise Exception("'sel' must return only one True")
    date = series[sel].index.strftime('%m/%Y').astype(str)
    value = round(series[sel], 2)
    out = dict()
    for (date, value) in zip(date, value): out['idx'] = date; out['val'] = value
    return out

class lm:

    def __init__(self, data, y_var):
        # data prep
        self.data = data.dropna()
        self.y = self.data[[y_var]].to_numpy()
        self.x = self.data.loc[:, self.data.columns != y_var].to_numpy()

        # split data
        self.x_train, self.x_test, self.y_train, self.y_test = train_test_split(
            self.x, self.y, test_size = 0.25)

        # model evalution
        self.lm = LinearRegression().fit(self.x, self.y)
        self.lm_train = LinearRegression().fit(self.x_train, self.y_train)

        # metrics
        self.y_pred = self.lm.predict(self.x)
        self.y_pred_train = self.lm_train.predict(self.x_test)

        self.residuals = self.y_pred - self.y
        #self.residuals_train = self.y_pred_train - self.y_train

        self.coef = dict(
            coefficients=self.lm.coef_.tolist(),
            intercept=self.lm.intercept_.tolist())
        self.coef_train = dict(
            coefficients=self.lm_train.coef_.tolist(),
            intercept=self.lm_train.intercept_.tolist())

        self.r_sqr = self.lm.score(self.x, self.y)
        self.r_sqr_train = self.lm_train.score(self.x_test, self.y_test)

        self.mae = mean_absolute_error(y_true=self.y, y_pred=self.y_pred)
        self.mae_train = mean_absolute_error(
            y_true=self.y_test, y_pred=self.y_pred_train)

        self.mse = mean_squared_error(y_true=self.y, y_pred=self.y_pred)
        self.mse_train = mean_squared_error(
            y_true=self.y_test, y_pred=self.y_pred_train)

        self.rmse = mean_squared_error(
            y_true=self.y, y_pred=self.y_pred, squared=False)
        self.rmse_train = mean_squared_error(
            y_true=self.y_test, y_pred=self.y_pred_train, squared=False)

    def params(self, train=True, deep=True):
        if train: return self.lm_train.get_params(deep=deep)
        else: self.lm.get_params(deep=deep)

    def predict(self, x, train=True):
        x = pd.DataFrame(x).to_numpy()
        if train: out = self.lm_train.predict(x)
        else: out = self.lm.predict(x)
        return pd.DataFrame({'pred': out[:,0]})

2.1. IPCA versus CDI¶

Agora fazendo algumas visualizações, fica fácil de perceber que as variações da inflação parecem mais erráticas e imprevisíveis, enquanto que a taxa de juros parece seguir um curso.

A explicação para o isto é que a taxa de juros CDI sempre acompanha a SELIC definida pelo Conselho Monetário Nacional, por se tratar de uma taxa definida artificialmente, fica fácil prevê-la. Já a inflação IPCA é definida dinâmicamente pelo mercado, e é apenas medida pelo IBGE, que não tem poder de influência algum sobre ela.

2.2. Encontrando os juros reais¶

Adicionando juros real, usando a fórmula de Fisher:

$r=\frac{(1+i)}{(1+\pi)}-1$

Onde $i$ são os juros nominais, $\pi$ a inflação, e $r$ os juros reais.

In [44]:
df.plot(kind='scatter', x='inflac_acm', y='juros_acm');

Muito interessante como estas duas variáveis parecem estar relacionadas! Mas existem algumas coisas que explicam este fato:

Uma delas é a forma com estas séries foram construídas. Você deve lembrar que no tópico 2.3 eu usei a função cumsum() para construir ambas as funções. Uma maneira de entender o que esta função faz, é que ela "emplilha" o próximo valor em cima da pilha de todos os outros valores anteriores, então pelo simples fato de a maioria dos valores das séries originais serem positivos (como vimos em 2.1), temos uma linha crescente como resultado em ambos os casos.

Mas outra coisa que podemos usar como explicação também é a função da taxa base de juros. Como eu disse anteriormente, o CDI sempre acompanha a SELIC, e o Conselho Monetário usa essa taxa para controlar a inflação, consequentemente, ele deve reagir ao nível atual de inflação definindo uma nova taxa de juros (juros altos em tempos de inflação alta). Como resultado, temos uma curva de juros real modesta, mas na maior parte do tempo positiva (como vimos em 2.2).

3.1. Tempo de resposta da inflação¶

Agora que sabemos como a inflação e a taxa de juros estão relacionadas entre si, uma coisa interessante que podemos fazer é tentar medir quanto tempo deve passar até que a inflação comece a responder à mudança da taxa de juros.

A maneira simples de fazer isso é atrasar uma série em relação à outra até que uma delas até conseguir a maior correlação possível. O tempo de atraso aplicado será o tempo de reação esperado para uma mudança na taxa de juros.

3.2. Prevendo a inflação futura com os juros presentes¶

Agora que já temos uma boa idéia de quanto tempo será necessário para que a inflação reaja à taxa de juros, podemos tentar prever a inflação nos próximos meses!

Você já deve estar cansado desse texto, mas o que temos no final vai compensar a leitura. Já que a taxa de juros de meses atrás pode ajudar a entender a inflação de hoje, podemos usar a taxa de juros de hoje para tentar entender a inflação do futuro! Vamos começar treinando um modelo de regressão, o nosso modelo terá este formato:

$$ \pi_{(t=14)} = \alpha \times i_{t=0} + \beta $$

Significa que usaremos os estimadores taxa de juros $i$ no momento $t=0$ para prever a inflação $\pi$ no momento $t=14$. $\alpha$ e $\beta$ vão nos ajudar a ajustar a curva de regressão no lugar certo, estes são os nossos parâmetros.

O motivo de usar a inflação adiantada, é ter a oportunidade de treinar o modelo com os dados mais recentes de inflação, o que é muito importante quando se trata de realizar previsões para o futuro (forecasting).
